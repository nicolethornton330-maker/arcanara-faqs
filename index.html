What is Arcanara? Getting Started

Arcanara is a tarot-focused Discord bot built for fast, readable card meanings and guided self-reflection. It supports slash commands for pulls and interpretations, and can include card images when permissions allow.

Arcanara isn’t here to “tell your future.” It’s designed to help you think clearly, notice patterns, and explore options.

How do I see what commands are available? Getting Started

Use Discord’s slash-command menu (type /) and look for Arcanara’s command list. Most users start with /meaning for specific cards and /cardoftheday for a daily pull.

How does /meaning work? Commands

/meaning returns upright and reversed meanings for a requested card. If image permissions are enabled, Arcanara may include the card image attachment alongside the text.

Tip: you can usually type partial card names and choose from Discord’s suggestions if autocomplete is enabled.

How does /cardoftheday work? Commands

/cardoftheday gives you a daily card pull intended to guide reflection for the day. It’s designed to be stable for that day (so you’re not “rerolling reality” every five minutes).

If your deck data changes in the background (e.g., card naming updates), Arcanara may fall back gracefully and re-draw to avoid errors.

Are Arcanara responses always ephemeral? Are they always embeds? Behavior

Ephemeral means “only you can see it” in Discord. Ephemeral responses can be plain text, embeds, or a mix — it depends on the command and what it’s returning.

Some commands may respond publicly by design (for shared readings or server features). If you’re unsure, check the command description or ask in the support server.

Why can’t I see card images? Images

Card images require the bot to have Attach Files in that channel. If Arcanara uses embeds for rich formatting, Embed Links may also be needed.

Fix: check channel permission overrides for the bot role, and confirm “Attach Files” is allowed where you’re using the commands.

Which permissions does Arcanara need? Permissions

For the channel where you use Arcanara, these are the usual requirements:

  • View Channel
  • Send Messages
  • Read Message History (helps with interaction updates and context)
  • Embed Links (if embeds are used)
  • Attach Files (for card images)

If commands respond but output is missing (especially images), it’s almost always a channel override issue.

Does Arcanara store my reading history? Privacy

Arcanara supports opt-in history for certain features (for example, tracking daily cards or lookups). If you haven’t opted in, history-related logging is intended to remain disabled.

If you want to enable or disable history tracking, use the bot’s relevant settings command (or ask in the support server for the current command name).

What data does Arcanara store? Privacy

Arcanara stores the minimum configuration required to function (like server-level settings, if enabled) and any user-specific settings you choose to save (such as preferences/tone).

Any history-style tracking is intended to be opt-in. If you need a formal privacy policy for Top.gg, request the current policy text in the support server.

Is Arcanara “advice”? How should I use readings responsibly? Guidance

Use Arcanara for reflection and decision support, not as a substitute for professional guidance. Tarot can be meaningful, but it’s not a license to ignore medical, legal, or financial reality.

A practical approach: treat readings as prompts — “What am I not noticing?” “What tradeoff am I avoiding?” — then make grounded choices with real-world information.

The command says “Interaction Failed” or Arcanara doesn’t respond. Why? Troubleshooting

Common causes:

  • Missing channel permissions (Send Messages / Embed Links / Attach Files)
  • Discord API hiccups (it happens)
  • Command timing out due to heavy load or a transient error

If it’s persistent, capture a screenshot and share it in the support server along with the command you ran and the channel name.

“I searched the deck but found no card named …” What should I do? Troubleshooting

That means Arcanara couldn’t match the card name you entered to the deck’s canonical card list. Try a shorter partial name (e.g., “lovers”, “high priestess”), or select from the autocomplete list if it appears.

If a commonly-used card name isn’t matching, report it — that’s usually a normalization/mapping fix.

How do I report a bug or request a feature? Support

Use the support server and post in the appropriate channel (support / bugs / suggestions). For bugs, include what you ran, what you expected, and what happened — screenshots or logs are gold.

Support inquiries receive a response within 24 hours or less.
